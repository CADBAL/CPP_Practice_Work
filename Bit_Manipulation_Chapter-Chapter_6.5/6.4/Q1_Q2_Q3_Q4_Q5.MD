# Easier to start from right-most end and then add all the values up

# (1 * 1) + (0 * 2) + ( 1 * 4) + (1 * 8) + (0 * 16) + (0 * 32) + (1 * 64) + (0 * 128)

# = 77. Answer is 77.

# Q2) Convert 93 to an 8-bit unsigned binary number, using both methods 1 and 2 above.

# Method 1:

# 93 / 2 = 46 R1
# 46 / 2 = 23 R0
# 23 / 2 = 11 R1
# 11 / 2 = 5 R1
# 5 / 2 = 2 R1
# 2 / 2 = 1 R0
# 1 / 2 = 0 R1

# Question asks for 8-bit unsigned binary number so extra zero is required for padding in front
# Adding leading zeros does not change the value of the binary number

# Answer is 0101 1101 (add the leading zero then write the remainders from the bottom up)

# Method 2:
# The largest  power of 2 smaller than 93 is 64

# 93 >= 64 True so (1). 93 - 64 = 29
# 29 >= 32 False so (0)
# 29 >= 16 True so (1). 29 - 16 = 13
# 13 >= 8 True so (1). 13 - 8 = 5
# 5 >= 4 True so (1). 5 - 4 = 1
# 1 >= 2 False so (0)
# 1 >= 1 True so (1). 1 - 1 = 0 Since 1 is lowest value divisible by 2 there are no more bit positions.

# if let's say it was 2 >= 2, 2 - 2 = 0 there would be one more remaining bit value of 0,
# same thing with 4 >= 4, 4 - 4 = there would be two more remaining bit values of zero.

# 0101 1101 = (1 * 0) + (0 * 2) + (1 * 4) + (1 * 8) + (1 * 16) + (0 * 32) + (1 * 64) + (0 * 128) = 93
# Added an extra zero bit at left-most side because it is needed because question asked for 8 bit unsigned
# binary number
# Therefore, the answer is proven correct.
