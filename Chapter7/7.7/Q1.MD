# What’s the difference between a variable’s scope, duration, and linkage? What kind of scope, duration, and linkage do global variables have?

# A variable's scope signifies when that file can be accessed in a program. The duration are the rules that signify when it is created and destroyed. The linkage signifies whether that file can be used in another file or not.

# Global variables have global scope, also known as file scope. 
# which means they can be accessed from the point of declaration to the end of the file in which they are declared.

# Global variables have static duration, they are created when a program starts and is only destroyed when the program ends. They always exist in memory.

# The linkage global variables have depends on the type of global variable and whether you specify they are "internal" or "external".
# Static "storage class specifier" means they can't be accessed outside the file they are defined.
# External means they can be forward declared outside the file they are defined.
